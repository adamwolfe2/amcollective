## Conventions I should keep using

- **One commit per logical change.** Small commits beat amend-spam.
- **Atomic PRs**: never mix marketing-site changes with backend changes.
- **No console.log in committed code** (per global rules).
- **Drizzle, not Prisma.** Already enforced in repo.
- **Clerk auth everywhere except `/api/mcp` and the public allowlist in middleware.ts.**
- **Always check `.env.local` and `.env.example` for env-var names before inventing new ones.**

## Conventions I should AVOID

- Don't run `fly launch` from a directory containing a Next.js project — it will pick up the wrong codebase. Always `cd ~/hermes-fly` first.
- Don't add `min_machines_running=1` services to Modal — it's not idle-free for Slack Socket Mode (lesson learned, ~$45/mo).
- Don't try to use Hermes' built-in HTTP mode for Slack — it's Socket Mode only. Forking Hermes was rejected.
- Don't trust Vercel auto-deploy as a deploy verification — Sentry CLI failures still mark the build as Error and prevent the production alias from promoting. Always check `vercel ls amcollective --prod` for `Ready`.
